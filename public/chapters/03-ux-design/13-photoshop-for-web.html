Chapter 13

Photoshop for Web

Document setup, asset export, GIF animation, and realistic UI effects

Photoshop is no longer the primary tool for web UI design (Figma is), but you will still encounter PSD files from legacy projects, marketing teams, and specialized asset creation. This chapter covers the subset of Photoshop you need: setting up documents, exporting web-optimized assets, creating GIF animations, and producing realistic UI effects. The goal is self-sufficiency — open a PSD, extract what you need, produce web-ready output without asking a designer.

Document Setup for Web

For web mockups, create documents at 1x resolution (72 PPI) at common screen widths:

  • Desktop: 1440×900 or 1920×1080
  • Tablet: 768×1024 or 1024×1366
  • Mobile: 375×812 (iPhone) or 390×844 (modern Android)

Use a 12-column grid (~60px columns with 20px gutters at 960px) — the standard used by Bootstrap, Tailwind, and most design systems. Set guides via View → New Guide.

Layer Hygiene

Well-organized PSDs follow the same principles as well-organized code:

  • Name every layer — "Rectangle 1 copy 5" is useless
  • Group related layers into folders (Header, Hero, Footer)
  • Use Smart Objects for reusable elements — edit once, update everywhere

Designing UI Mockups

You do not need to be a designer to produce clean mockups. Focus on structure and spacing — visual flair is what CSS is for.

Shape Layers for UI Elements

Use shape layers (rectangle, rounded rectangle, ellipse) rather than selections or brushes. Shape layers are vector — they scale without quality loss and can be exported as SVG. Rounded corners for buttons: 4-8px for standard buttons, 20px+ for pill shapes.

Component Libraries in Photoshop

Design your UI as reusable components, just like you would in React. A button designed once should be copied (not redrawn) wherever it appears. If the button design changes, update the master and replace all instances. This is the Photoshop equivalent of DRY — and most designers do not do it, which is why design handoff is often messy.

Layer Organization

  • Name every layer — "Rectangle 1 copy 5" is useless
  • Group related layers into folders (Header, Hero, Footer)
  • Use color labels for layer groups (red = UI, blue = images, green = text)
  • Use Smart Objects for reusable elements — edit once, update everywhere

Exporting Web Assets

A poorly exported asset degrades a site more than any CSS mistake. Use File → Export → Export As (not the legacy "Save for Web") — it supports modern formats and generates multiple sizes at once.

Format Selection

Photos / gradientsJPG (quality 80-85)No transparency. Start at 80% quality and reduce until unacceptable.
Screenshots / UI with transparencyPNG-24 → PNGQuantLossless transparency. PNGQuant reduces file size 60-80% with no visible loss.
Flat graphics / logosSVGVector — infinite resolution, tiny file size. Export from the source file.

Retina Export

Export UI assets at 2x the display size and use CSS to display at 1x. A 100×40px button → export at 200×80px → set width: 100px; height: 40px in CSS. This produces sharp output on Retina displays.

SVG Over PNG When Possible

SVG scales to any size without quality loss and produces tiny files for simple graphics. Logos, icons, and illustrations should always be SVG. Only use PNG when the asset contains photographic content or complex gradients.

Creating GIF Animations

GIF animation in Photoshop works by cycling through slightly different frames. The key principle: inconsistency creates realism.

Workflow

  1. Prepare. Duplicate the background layer and lock the original as a safety net.
  2. Isolate. Select the element to animate (Lasso or Quick Selection). Convert to its own layer (Ctrl+J).
  3. Open Timeline. Window → Timeline → "Create Frame Animation".
  4. Build frames (3-10). Duplicate the first frame. On each new frame, make tiny changes: nudge position 1-2px, vary opacity 60-100%, apply subtle Gaussian blur (0.3-0.8px) on some frames but not others.
  5. Set timing. Select all frames → set delay to 0.1 or 0.15 seconds. Vary timing between frames for uneven motion.
  6. Loop. Set to "Forever".
  7. Export. File → Export → Save for Web (Legacy) → GIF format. Reduce color palette to 128 or 64 colors for smaller file size.
Why Small Variations Matter

The human eye detects patterns. If a flame moves exactly 2px on every frame with perfect opacity, it looks mechanical. Real motion has randomness — slight timing differences, subtle blur changes, tiny position inconsistencies. The less uniform, the more natural.

Embossed Effects on Textures

Making text or a logo look pressed into a surface (leather, paper, metal) is a common design request. The key insight: real embossing is surface deformation plus light coupling — not just shadow styling.

Workflow

  1. Prepare. Place the logo/text above the background texture. Convert to Smart Object.
  2. Bevel & Emboss. Layer Style → Bevel & Emboss. Style: Inner Bevel. Direction: Down (for deboss). Depth: 150-300%. Highlight: Screen at 10-30% opacity. Shadow: Multiply with a dark tone matching the texture.
  3. Blend If. In the Layer Style panel, adjust the "Underlying Layer" sliders. Hold Alt to split sliders for a smooth gradient transition.
  4. Displace. Filter → Displace using a grayscale version of the texture as the map. This physically warps the logo to follow the texture's bumps — the single most important step for realism.
  5. Final polish. Gaussian blur (0.3-0.8px) to soften vector edges. Add noise (1-3%) to match grain. Reduce layer opacity to 70-90% so texture shows through.
Why Most Embossing Looks Fake

Most designers stop at step 2 — Bevel & Emboss alone creates a sticker-on-top effect. Realism comes from steps 3-5: blending edges into the texture, physically warping to match surface irregularities, and matching noise/grain. Real pressed-in objects deform the surface around them.

Junior Developer

How a Junior Might Think

  • Exports everything as massive PNGs at wrong resolution
  • No organized layer structure — cannot find anything in the PSD
  • Thinks Bevel & Emboss alone is "good enough" for embossing
  • Creates GIFs with uniform frame motion that looks mechanical
Senior Developer

How a Senior Thinks

  • Exports SVG for vectors, WebP for photos, 2x PNG for Retina UI
  • Maintains organized PSDs with named, grouped layers
  • Uses displacement maps, Blend If, and noise matching for realistic textures
  • Builds GIFs with deliberate frame-to-frame inconsistency for natural motion
