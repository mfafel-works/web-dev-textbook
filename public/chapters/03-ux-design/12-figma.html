Chapter 12

Figma for Web

Frames, auto layout, components, prototyping, and developer handoff

Figma is the industry-standard design tool for web interfaces. It runs in the browser, supports real-time collaboration, and produces output that developers can inspect directly. This chapter covers the subset of Figma you need to design and hand off web interfaces — not every feature, but every feature a web developer will encounter.

Frames vs Groups

This is the most fundamental concept in Figma and the source of most confusion for new users.

FrameGroup
What it isA container with its own layout spaceA selection of objects bundled together
ShortcutF or ACtrl+G

ExportCan have its own export settingsExports as merged image
When to useAlways. Frames are the building blocks of UI design.Rarely. Only for temporary grouping during editing.

Rule: use frames, not groups. Groups are static. Frames are structural. Every screen, every component, every card should be a frame. This is the single habit that separates organized Figma files from chaotic ones.

Frames Are Like HTML Elements

Think of frames as <div>s. A group is like wrapping elements in a <span> — visually grouping them without structural meaning. Frames let you control layout (auto layout = flexbox), clipping, and constraints. If you want to build a page, you should almost never use the Group command.

Auto Layout

Auto Layout is Figma's equivalent of CSS Flexbox. It makes frames automatically adjust to their content — buttons grow with text, cards resize with content, spacing is consistent.

How Auto Layout Works

Select a frame and press Shift+A to apply auto layout. You get three controls:

  • Direction — horizontal (row) or vertical (column). Matches flex-direction.
  • Spacing between items — the gap between children. Matches CSS gap.
  • Padding — inner space on all four sides. Matches CSS padding.
Auto layout → CSS mapping
/* Figma auto layout: horizontal, gap 16, padding 24 */
/* maps to: */
.container {
  display: flex;           /* auto layout on */
  flex-direction: row;    /* horizontal */
  gap: 16px;              /* spacing between items */
  padding: 24px;          /* inner padding */
}

/* Figma: auto layout with "Wrap" enabled */
/* maps to: */
.container {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}

Design in Auto Layout from the Start

Manually positioning elements in Figma (click-drag placement) is like using position: absolute for everything. It works initially but breaks the moment anything changes. Auto layout keeps your design flexible — add a new button, and everything shifts automatically. Design with auto layout first, manually position only when necessary.

Components and Variants

Components are reusable design elements. A button designed once as a component can be used hundreds of times. Change the master component, and all instances update.

Component Workflow

  1. Create a frame with your element (e.g., a button)
  2. Select it and press Ctrl+Alt+K (or click the diamond icon in the toolbar) to create a component
  3. Use the Assets panel to drag instances of your component onto other frames
  4. Edit the master — all instances reflect the change

Overrides

Instances can override specific properties (text, fill, size) without detaching from the master. Changing the text of a button instance to "Submit" does not affect other instances. This is the power of components: shared structure, instance-specific content.

Variants

Variants group related components into a single component set. A button might have variants: Primary, Secondary, Ghost, Danger. Each variant can have different fills, border, and text styles, but they are all the same component.

Button component set: ├── Variant: Primary (blue bg, white text) ├── Variant: Secondary (white bg, blue border) ├── Variant: Ghost (transparent, blue text) └── Variant: Danger (red bg, white text) Each variant also has states: Default, Hover, Pressed, Disabled.

To create variants: select multiple components → right-click → "Combine as variants". Define variant properties in the right panel (e.g., "Type", "Size", "State").

Constraints and Resizing

Constraints determine how a child element behaves when its parent frame resizes. In the right panel, each layer has horizontal and vertical constraint controls.

Left + Top (default)Stays fixed relative to top-leftposition: static

For children inside auto layout frames, constraints are replaced by resizing settings: "Fill container" (width/height 100%) or "Hug contents" (shrink-wrap). Auto layout's resizing is more powerful and predictable than manual constraints.

Prototyping

Figma's prototyping mode connects frames to simulate user flows. Select an element, click the "Prototype" tab in the right panel, and drag the arrow to the destination frame.

Smart Animate

When two frames share layers with the same name, Figma's Smart Animate interpolates the transition. Use this for list item reordering, menu transitions, and page state changes. Set the animation to "Smart Animate" with a 200-300ms ease-out transition for natural-feeling motion.

Prototypes Are Not Production

A Figma prototype simulates interactions, but the actual implementation may differ. Scroll behavior, real data loading, keyboard navigation, and edge cases do not exist in prototypes. Use prototypes to validate flows and get sign-off — not as a pixel-perfect specification for every interaction state.

Styles and Design Tokens

Figma styles are reusable design values — exactly like CSS custom properties. Define colors, text styles, effects, and grid styles once, apply them everywhere.

Creating Styles

  • Color styles — Select a layer → Click the color fill → "Create style". Name it Primary/Default, Neutral/100, etc.
  • Text styles — Select a text layer → Click "Create style" in the text section. Name it Heading/H1, Body/Default, etc.
  • Effect styles — Save shadow or blur configurations as reusable effects.
Figma style naming → CSS token mapping
/* Figma style: "Primary/Default"       → --color-primary: #2563eb */
/* Figma style: "Primary/Hover"        → --color-primary-hover: #1d4ed8 */
/* Figma style: "Neutral/100"          → --color-neutral-100: #f1f5f9 */
/* Figma style: "Neutral/700"          → --color-neutral-700: #334155 */
/* Figma style: "Heading/H1"           → --text-h1: 2.5rem/700 */
/* Figma style: "Body/Default"         → --text-body: 1rem/400 */

Naming conventions matter. Use a slash-separated hierarchy: Category/Property/Variant. This creates organized menus in the style picker.

Sync Styles with Code

Your Figma color palette should match your :root CSS custom properties exactly. When the designer updates "Primary/Default" in Figma, you should update --color-primary in CSS. Discrepancies between design and code are a leading source of "it doesn't match the mockup" bugs. Some teams use plugins (Design Tokens, Figma Tokens) to export tokens directly.

Developer Handoff

Figma's developer handoff features let developers inspect designs without asking designers for specs.

The Inspect Panel

Select any layer in a Figma file shared with you as a developer. The right panel shows an "Inspect" tab with:

  • Dimensions and position — width, height, x, y coordinates
  • CSS properties — auto-detected CSS including flexbox, fonts, colors, shadows
  • Assets — export SVG or PNG at different resolutions
  • Spacing — margin and padding measurements
Figma Inspect panel output
/* Auto-detected CSS from Figma */
.button {
  display: flex;
  align-items: center;
  justify-content: center;
  padding: 12px 24px;
  gap: 8px;
  background: #2563eb;
  border-radius: 8px;
  color: #ffffff;
  font-family: Inter, sans-serif;
  font-size: 16px;
  font-weight: 500;
  line-height: 1.5;
}

Important: Figma's auto-generated CSS is a starting point, not production code. It may use absolute positioning where you should use flexbox, or miss responsive considerations. Use it as reference, not a copy-paste source.

Exporting Assets

For images and icons that are not CSS-reproducible:

  1. Select the layer
  2. In the right panel, click "+" in the Export section
  3. Choose format: SVG for vectors/icons, PNG for raster images (2x for Retina)
  4. Set suffix: @2x for 2x exports
Export at the Right Resolution

Always export critical UI elements (icons, logos) as SVG — they are resolution-independent and small. For raster images, export at 2x (e.g., 200x200 image exported at 400x400) and use width: 100px in CSS. This ensures clarity on Retina displays.

Junior Developer

How a Junior Might Think

  • Uses groups instead of frames — file becomes unmanageable
  • Positions everything manually — moves break when anything changes
  • Does not use components — changes need to be repeated 20 times
  • Does not use styles — colors and fonts are inconsistent
  • Relies entirely on Figma's auto-generated CSS instead of reasoning about layout
  • Does not use the Inspect panel — asks designer for measurements
Senior Developer

How a Senior Thinks

  • Uses frames with auto layout for everything — designs are flexible and maintainable
  • Creates components for repeated patterns — updates propagate automatically
  • Defines color and text styles — keeps the design consistent
  • Understands that Figma's auto-generated CSS is a hint, not a spec
  • Uses the Inspect panel to check spacing, colors, and fonts efficiently
  • Keeps Figma styles and CSS tokens in sync
