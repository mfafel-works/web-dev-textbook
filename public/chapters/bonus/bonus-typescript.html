TypeScript Basics

Why type safety matters, basic types, interfaces, and adding TS to your workflow

TypeScript is JavaScript with type annotations. It catches bugs before your code runs — instead of discovering a bug in production, you see it in your editor while writing. This bonus covers the TypeScript essentials you need to add type safety to your projects.

Why TypeScript?

JavaScript is dynamically typed — types are determined at runtime. This flexibility is powerful but leads to common errors:

JavaScript: runtime error
// This looks fine but fails at runtime
function greet(user) {
  return "Hello, " + user.name.toUpperCase();
}

greet(42); // TypeError: Cannot read property 'name' of 42

TypeScript catches this at compile time:

TypeScript: compile-time error
function greet(user: { name: string }) {
  return "Hello, " + user.name.toUpperCase();
}

greet(42); // Error: Argument of type 'number' is not assignable to parameter of type '{ name: string; }'
Compile Time vs Runtime

Runtime errors happen when your code is already running — users see them. Compile-time errors happen when you build — you see them in your editor before deployment. TypeScript moves more errors to the "you see it immediately" category.

Basic Types

TypeScript adds type annotations to JavaScript. Here are the core types:

Basic type annotations
// Primitive types
let name: string = "Alice";
let age: number = 30;
let isActive: boolean = true;

// Arrays
let users: string[] = ["Alice", "Bob"];
let scores: number[] = [95, 87, 92];

// Objects
let user: { name: string; age: number } = {
  name: "Alice",
  age: 30
};

// Arrays of objects
let products: { id: number; name: string; price: number }[] = [
  { id: 1, name: "Widget", price: 29.99 },
  { id: 2, name: "Gadget", price: 49.99 }
];

Type Inference

TypeScript often figures out the type automatically — you don't always need to write annotations:

Type inference
// TypeScript knows this is a string
let message = "Hello"; // type: string

// TypeScript knows this is a string array
const colors = ["red", "blue", "green"]; // type: string[]

// Functions return types are inferred
function add(a: number, b: number) {
  return a + b; // return type: number
}

When to use explicit types: function parameters, function returns, complex object shapes, API responses.

Interfaces

Interfaces define the shape of objects — reusable type definitions:

Interfaces
interface User {
  id: number;
  name: string;
  email: string;
  isAdmin?: boolean; // ? makes it optional
}

// Use the interface
function greetUser(user: User) {
  return `Hello, ${user.name}!`;
}

greetUser({ id: 1, name: "Alice", email: "alice@example.com" });
greetUser({ id: 2, name: "Bob", isAdmin: true });

Utility Types

TypeScript provides built-in utility types that transform types:

Utility TypeWhat It DoesExample
Partial<T>All properties optionalPartial<User>
Required<T>All properties requiredRequired<User>
Pick<T, K>Select specific propertiesPick<User, 'id' | 'name'>
Omit<T, K>Exclude specific propertiesOmit<User, 'email'>

Getting Started

Add TypeScript to an existing project:

Adding TypeScript to a project
# Install TypeScript
npm install --save-dev typescript

# Create tsconfig.json
npx tsc --init

# Check for errors
npx tsc

# Run in watch mode
npx tsc --watch
Start Gradually

You don't need to convert everything at once. Start by adding .ts files alongside your .js files — TypeScript understands JavaScript, so you can migrate incrementally.
