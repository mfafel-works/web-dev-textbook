How Hash Routing Works

Hash routing uses the fragment identifier (#/path) in the URL. The browser never sends the hash fragment to the server, so the server always serves index.html. JavaScript reads window.location.hash and renders the corresponding page. Try clicking links or typing a path below.

Without Hash Routing

Each page is a separate HTML file. The user sees real URLs like /about.html, but navigation causes a full page reload. Animations break, state is lost.

With Hash Routing

A single HTML file serves all pages. URLs like /#/about never hit the server. Navigation is instant — no reload, no state loss. The tradeoff: URLs are slightly uglier, but deployment is trivial (no server config needed).
