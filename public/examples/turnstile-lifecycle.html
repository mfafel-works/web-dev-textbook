Turnstile Token Lifecycle

Interactive walkthrough of how Cloudflare Turnstile generates, validates, and expires tokens in an SPA.

What Happens During Turnstile Verification

Turnstile is a privacy-preserving CAPTCHA replacement. When the widget renders, it runs a browser challenge (usually invisible). On success, it generates a one-time token that must be verified server-side. The token expires after 300 seconds (5 minutes) — a common SPA gotcha.

Common SPA Mistake

Rendering the Turnstile widget once and reusing the same form HTML across navigations. When the user navigates away and back, the token is stale — the server rejects it. The user sees a generic error and doesn't know why.

Correct SPA Approach

Each navigation re-renders the page content, which destroys and re-creates the Turnstile widget. This forces a fresh challenge and a new token. Alternatively, call turnstile.reset() on the widget container when reusing cached DOM.

Server-Side Verification Code (PHP) 
function verifyTurnstile($token): bool {
    if (empty($token)) return false;

    $ch = curl_init('https://challenges.cloudflare.com/turnstile/v0/siteverify');
    curl_setopt_array($ch, [
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_POSTFIELDS     => [
            'secret'   => $_ENV['TURNSTILE_SECRET_KEY'],
            'response' => $token,
        ],
    ]);

    $result = json_decode(curl_exec($ch), true);
    return ($result['success'] ?? false) === true;
}
