FormData vs JSON Encoder

See how the same form data looks in both encoding formats — and why choosing the wrong one silently drops file uploads.

The Gotcha

JSON.stringify() cannot serialize File objects. If your form has a file input and you extract values via new FormData(form) then send as application/json, the file is silently dropped. Use FormData directly (with no manual Content-Type) for file uploads, or JSON for text-only payloads — never mix the two.

Common Mistake

Trying to send a file via JSON:

const fd = new FormData(form);
const data = { name: fd.get('name') };
data.attachment = fd.get('file');
await fetch('/api', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(data)  // file is lost!
});

The File object becomes {} in JSON.

Correct Approach

Sending a file via FormData:

const fd = new FormData(form);
await fetch('/api', {
  method: 'POST',
  // Do NOT set Content-Type —
  // browser sets multipart/form-data
  // with the correct boundary
  body: fd
});

File data is included. Browser handles encoding.
